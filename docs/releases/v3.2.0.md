## What's New

### CI/CD Enforcement Pipeline

```bash
# Gate your PRs on policy violations
curl -H "Authorization: Bearer $IDE_API_KEY" \
  -X POST http://localhost:8081/api/v1/policy/check \
  -H "Content-Type: application/json" \
  -d '{"input": "rm -rf /", "categories": ["bash"]}'
```

Returns pass/fail with detailed violations including rule ID, severity, line number, and column position.

### Webhook Notifications

5 new MCP tools for managing webhook endpoints:

```
configure_webhook  тЖТ Create/update webhook with HMAC secret
test_webhook       тЖТ Send test event
list_webhooks      тЖТ View all webhooks for a team
delete_webhook     тЖТ Remove a webhook
get_webhook_deliveries тЖТ View delivery history
```

Events delivered: `violation.detected`, `halt.triggered`, `budget.exceeded`

### Token Budget Ledger

5 new MCP tools for managing AI spend:

```
configure_budget   тЖТ Set token/cost limits per team/model
get_budget_status  тЖТ Current spend vs limits
list_budgets       тЖТ All budget configs for a team
get_budget_history тЖТ Usage history with token/cost totals
delete_budget      тЖТ Remove a budget config
```

Supports Claude 3.5 Sonnet, Claude 3 Opus/Haiku, GPT-4o, GPT-4-turbo, GPT-3.5-turbo, local-llama.

### Agent Lifecycle State Machine

5 new MCP tools for managing agent phases:

```
create_agent_session    тЖТ Start agent in idle state
transition_agent_state  тЖТ Move to next phase (validated)
get_agent_state         тЖТ Current state + transition history
list_agent_sessions     тЖТ All sessions for a team
force_agent_state       тЖТ Admin override (requires justification)
```

Valid lifecycle: `idle тЖТ planning тЖТ active тЖТ review тЖТ release тЖТ idle` (with `halted` as emergency stop)

## Security Fixes (8 total)

| Issue | Severity | Fix |
|-------|----------|-----|
| SSRF via `os.ReadFile` (4 vectors) | High | Added `safeReadFile()` with path traversal validation |
| ReDoS in web handler | High | Exported `validation.CompilePattern()`, replaced raw `regexp.Compile` |
| Unauthenticated POST endpoints | Medium | `/api/ingest`, `/api/updates/check` now require auth |
| File extension bypass (`.json`) | Medium | Removed from public asset allowlist |
| Dead halt condition | High | `< 0` тЖТ `> 0` (halt mechanism was completely non-functional) |
| Duplicate function | Medium | Removed duplicate `buildToolResult` from `mcp/server.go` |
| Weak session tokens | Medium | 64-bit тЖТ 192-bit |
| Secrets scanner gaps | Low | 7 тЖТ 22 patterns (GCP, Azure, Stripe, npm, PyPI, HF, DB strings, SendGrid, Twilio, Mailgun) |

## Upgrading

```bash
# 1. Pull the latest code
git pull origin main

# 2. Update .env (add JWT_SECRET if missing)
cp .env.example .env  # Compare and merge changes

# 3. Run migrations
cd mcp-server && make migrate-up

# 4. Rebuild and restart
make compose-restart  # Docker Compose
# or
make build && systemctl restart guardrail-mcp  # Systemd

# 5. Verify
curl http://localhost:8081/health/ready
curl http://localhost:8081/docs  # API explorer
```
